Who can use this tool?

PMM and PSP are eligible to run this tool. PMM(Mandy Fan) runs site level PCBA allocation every Monday for PCBA site execution; PSP can rerun the allocation for respective BU during other days of the week upon needs.

What data are need to run the tool?

  1. 3a4 Backlog: download from UOV with view: kw_PA (default includes all DF sites and all BUs, user can limit to certain BUs if only do allocation for certain BUs);
  2. Supply data file (includes SCR, in-transit, DF OH, sourcing rules): download from "Datasource" tab via "Download supply data file from SCDx-POC)";
  3. Exceptional priority(optional): maintained in smartsheet by PSP as needed;
  4. TAN grouping(optional): maintained in smartsheet by PSP as needed;
  5. Missing in-transit (for SRG only): maintained in smartsheet as needed by Mandy as short term fix of SCDx missing data.

How PMM run the tool on Monday for whole site?

  1. Download 3a4 and supply data file;
  2. Upload the 2 files from the "Allocation" tab;
  3. Put in appropriate PCBA org;
  4. Select appropriate email option;
  5. Click "Make Allocation".

How PSP run the tool for responsible BU on other days?

  1. Download 3a4 and supply data file;
  2. Adjust the supply data file as needed - if updated supply not reflected in SCDx;
  3. Upload the 2 files from the "Allocation" tab;
  4. Put in appropriate PCBA org and BU;
  5. Select appropriate email option;
  6. Click "Make Allocation".

What is Exceptional priority?

PSP can give an exceptional priority to any SS to overwrite the priority setting in UOV. This can be used in situations like:
  1. Order is not reflecting as priority in UOV but PSP wants to consider it as priority;
  2. PSP may want to give higher priority to certain orders (e.g. L4, BUP) so they get absolute priority even though this may lead to other OTD hits if site execute accordingly;
  3. PSP has a close control on order priority and require site to execute strictly following a specific order list in sequence;
  4. Etc.
PSP can set the priority via this smartsheet. Default priority ranking#: PR1:1; PR2:2; PR3:3; L4:4; BUP:5; YE:6. Rank#<4 will have absolute priority surpassing FCD of other orders; Rank#>=4 will be prioritized on top of FCD dates.

What is Tan grouping?

Some BU have different TAN supply to different DF sites but which share the same materials at PCBA level, by grouping them we can not only decide the allocation but also let PCBA site know how to allocate the materials between the different TANs in production. PSP can set the grouping as needed via this smartsheet.
