What data are needed to run the tool?

  1. 3a4 Backlog: download from UOV (under view: PCBA_ALLOCATION - default includes all DF sites and all BUs, PSP can limit to certain BUs if only do allocation for certain BUs);
  2. Supply data (includes SCR, in-transit, DF OH, sourcing rules): read from SCDx API during allocation; can also download as file if needed;
  3. Exceptional priority(optional): maintained in smartsheet by PSP as needed to overwrite default priority;
  4. TAN grouping(optional): maintained in smartsheet by PSP as needed to group same material TANs;
  5. Exceptional sourcing split(optional): maintained in smartsheet by PSP as needed to correct the sourcing split;
  6. Missing in-transit (for SRG only): maintained in smartsheet as needed by Mandy as short term fix of SCDx missing data for cross partner intransit.

How to run the allocation?

  1. Download 3a4 from UOV (via view: PCBA_ALLOCATION) and upload to this tool;
  2. Supply data can be read from SCDx API directly during allocation run; or download from the API in advance - you can then manually adjust any data in the file if need to - and upload to this tool;
  3. Select PCBA org (mandatory), and BUs (optional);
  4. Click "Make Allocation".
  5. Download/Share the result from the "Download" tab.

What is the order ranking priority we use?

We follow general service 17 order priority guideline to rank the orders. This is the priority sequence we used: PR1/2/3/L4/BUP/OSSD/YE/QTY/REVENUE_NON_REVENUE.

What is Exceptional priority?

PSP can give an exceptional priority to a non-priority SS so they can also get an absolute high priority despite OSSD and reflect in allocation - this should be used only exceptionally. Default priority ranking#: PR1:1; PR2:2; PR3:3; L4:4; BUP:5. PSP can set exceptional priority via this smartsheet. E.g., if one SS is added with Ranking# 4.1, this SS will be ranked after L4; or if added with Ranking# greater than 5, they will be after BUP; etc.

What is Tan grouping?

Some BU have different TAN supply to different DF sites but which share the same materials at PCBA level, by grouping them we can not only decide the allocation between DF but also let PCBA site know how to allocate the materials between the different TANs to best optimize the production. PSP can set the grouping as needed via this smartsheet.

What is Exceptional sourcing split?

Some TAN have wrong sourcing split in SCDx intentionally due to scheduling would be impacted if not in some cases. PSP can put in a correct sourcing split via this smartsheet so the allocation will consider the right split.

Email sharing

Email sharing recipients is based on the subscription table based on relative ORG and BU. If allocation file is for specific BUs, only people indicated BU in the subscription will receive the email. Backlog detail data will be removed from the shared file, internal users can download from this site to view the full data.
